Quickstart

Four steps from zero to a fully managed research project.

1

Create Project & Import Template

Create a new project from the dashboard, then download the YAML template. Fill it with any LLM (GPT-4, Gemini, Claude) and upload the result.

# 1. Create project via UI or API
POST /api/v1/projects/import
{ "id": "my-project", "name": "My Research" }

# 2. Download template, fill with AI, upload
GET /api/v1/templates/project_content
POST /api/v1/projects/my-project/import_template
Overview, plans, steps, timeline auto-generated
2

Import Literature CSV

Upload a CSV export from Undermind or Zotero. Format is auto-detected; you can also select it manually from the dropdown.

# Upload CSV — format auto-detected
POST /api/v1/literature/projects/{id}/import_csv
Content-Type: multipart/form-data
file: papers.csv
csv_format: auto | undermind | zotero
Papers parsed, deduped, and indexed with tags
3

Track Execution

Cycle execution step statuses (todo → doing → done → blocked). Log progress with timestamped punch-in entries. Check off vibe coding and vibe research items.

# Update a step status
PATCH /api/v1/projects/{id}/execution/vc/{stepId}
{ "status": "doing" }

# Append a log entry
POST /api/v1/projects/{id}/logs/vibe_coding
{ "entry": "Implemented data loader" }
Progress bars update on Home dashboard
4

Generate Reports

Generate daily or weekly reports from real project data — checklist progress, logs, artifact additions, literature imports, and milestone status changes.

# Generate daily report
POST /api/v1/reports/daily

# Generate weekly report
POST /api/v1/reports/weekly

# Reports saved to reports/daily/ and reports/weekly/
Markdown reports with progress deltas and summaries

Features

Everything you need to manage research projects end-to-end.

◧

Portfolio Dashboard

Project cards with priority badges, RAG status, dual VC/VR progress bars, blocker counts, and literature metrics. Sort and filter across your entire research portfolio.

◉

Overview-First Project Pages

Structured project overview with section cards, accordions, milestone tables, and next-actions checklists. The single source of truth for each project.

◈

Literature Ingestion

Import CSV from Undermind or Zotero with auto-format detection. Search, tag, and annotate papers. Track citation counts and link papers to projects.

◆

Artifact Registry + Timeline

Track code repos, datasets, papers, demos, and reports with color-coded type badges. Milestones with deadlines, acceptance criteria, and kanban boards.

▤

Daily & Weekly Reports

Auto-generated reports derived from real project data — step progress, log entries, artifact additions, literature imports, and milestone status changes.

✦

AI Template Generation

Download a YAML template, fill it with any LLM, and upload. The system generates overviews, experiment plans, paper plans, execution steps, and timelines.

⚙

Multi-Provider Config

Configure AI providers including OpenAI, Anthropic, Google, and MiniMax. Per-provider API keys, model selection, and context window settings.

⇄

Execution Steps

Structured VC and VR execution steps with status cycling (todo → doing → done → blocked). Inline editing, acceptance criteria, and milestone linking.

⛁

Data Maintenance

Reindex literature, rebuild project caches, and export full data summaries. System health dashboard with data path inspection and integrity checks.

Frequently Asked Questions

What is Resonance?
Resonance is an open-source research project management dashboard designed for the "Vibe Coding + Vibe Research" workflow. It combines portfolio tracking, literature management, AI-assisted planning, execution step tracking, timeline/kanban boards, and report generation into a single self-hosted application.
What tech stack does it use?
The backend is Python 3.11+ with FastAPI and Pydantic V2. The frontend is React 18 + TypeScript + Vite + TailwindCSS v4 with React Query for data fetching. Data is stored as flat YAML/JSON/Markdown files — no database required.
How does the AI template workflow work?
Download a YAML template from any project page. Paste it into GPT-4, Gemini, Claude, or any LLM and ask it to fill all sections. Save the output as JSON, then upload it. The system auto-generates a structured overview, experiment and paper plans, VC/VR execution steps, milestones, and kanban items.
What literature CSV formats are supported?
Both Undermind and Zotero CSV exports are supported. The importer auto-detects the format from column headers (Zotero markers: Key, Item Type, Abstract Note; Undermind markers: Relevance Summary, Topic Match Score). You can also manually select the format from a dropdown.
Do I need a database?
No. All data is stored as flat files (YAML, JSON, Markdown, JSONL) in the project directory. This makes it easy to back up, version-control, and inspect your data with standard tools. The backend uses atomic file writes to prevent corruption.
Can I self-host it?
Yes — Resonance is designed to be self-hosted. Clone the repo, install Python and Node dependencies, build the frontend, and run the FastAPI server. It works on any Linux/macOS machine with Python 3.11+ and Node.js 20+.
What AI providers are supported?
The settings page supports configuring multiple AI providers including OpenAI, Anthropic, Google (Gemini), and MiniMax. Each provider has its own API key, base URL, and model list. The AI template workflow works with any LLM — you fill the template externally and upload the result.
How are reports generated?
Daily and weekly reports are generated from real stored data: checklist progress changes, punch-in log entries, artifact additions, literature imports, and timeline milestone status updates. Reports are saved as Markdown files and viewable from the Reports page with a built-in reader.
