01. Setup

Quickstart

Deploy locally and start tracking your research workflow in minutes.

install.sh
# 1. Clone the repository
$ git clone https://github.com/Mr-Tieguigui/vibeCR
$ cd vibeCR

# 2. Install dependencies (Requires Python 3.11+ & Node 20+)
$ pip install -r backend/requirements.txt
$ npm install --prefix frontend
Dependencies installed successfully
run.sh
# 1. Start the FastAPI backend
$ uvicorn backend.app.main:app --port 8007

# 2. Start the React frontend
$ npm run dev --prefix frontend

# Dashboard running at http://localhost:5179
Server running and UI ready
workflow.sh
# 1. Create a project using AI templates
> Upload filled JSON template

# 2. Ingest literature
> Upload Undermind/Zotero CSV

# 3. Track execution & generate reports
> Cycle tasks, log updates, export daily.md

06. Details

FAQ

What is "Vibe Coding"?
A workflow where you direct an AI agent (like Cascade or Aider) to write code, while you focus on architecture, testing, and steering. It relies on tight feedback loops and rigorous tracking.
Do I need an OpenAI or Anthropic API key?
No. VibeCR Monitor uses no backend AI APIs. You fill out project templates locally using any web UI (ChatGPT, Claude, Gemini) and upload the resulting JSON.
Where is my data stored?
All your data is stored locally in plain text files (YAML, JSON, Markdown) inside the `projects/` directory. No database is required, making it perfect for Git version control.
How does the literature importer work?
Export a CSV from Undermind or Zotero, and upload it via the VibeCR dashboard. The system auto-detects the source based on column headers and normalizes the data into a unified local index.
Can I self-host this on a remote server?
Yes. It's standard FastAPI + React. You can run it on a VPS and access it via the web, but it's designed primarily as a local desktop service.
